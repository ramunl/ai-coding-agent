# workflow.py

**Purpose**: Orchestrate the Git and GitHub workflow for implementing features and bugfixes.

## Overview
Manages branching, implementation execution, commits, pushes, and pull request creation.

## Branch Name Management

### `slugify_branch_name(change_description: str, prefix: str = "feature") -> str`
Converts a feature/bug description into a valid Git branch name.

**Process**:
1. Lowercase the description
2. Replace non-alphanumeric chars (except . _ / -) with hyphens
3. Remove consecutive hyphens
4. Trim leading/trailing hyphens, dots, slashes
5. Default to "change" if result is empty
6. Prefix with "feature/" or custom prefix
7. Limit to 80 chars after prefix

**Output Format**: `{prefix}/{slug}`

**Examples**:
- "Add offline support" → "feature/add-offline-support"
- "Fix crash on startup" → "bugfix/fix-crash-on-startup"
- "Implement @user/repo#123" → "feature/implement-user-repo-123"

### `validate_branch_prefix(prefix: str) -> None`
Validates branch prefix format.

**Allowed Characters**: A-Z, a-z, 0-9, . _ -

**Raises**: `ValueError` if prefix contains invalid characters

### `validate_branch_name(branch_name: str) -> None`
Validates complete branch name per Git naming rules.

**Checks**:
- Doesn't start or end with "/"
- Doesn't end with "."
- Doesn't contain ".."
- Doesn't contain "@{"
- Doesn't contain "\\"
- Only contains A-Z, a-z, 0-9, . _ / -

**Raises**: `ValueError` if validation fails

## Implementation Workflow

### `implement(plan: str, branch_name: str, agent: str | None = None) -> None`
Executes the selected implementation agent, Codex or Claude, to implement a feature based on a plan.

**Process**:
1. Validates branch name
2. Runs: `git checkout main` - Switch to main branch
3. Runs: `git pull origin main` - Get latest code
4. Runs: `git checkout -b {branch_name}` - Create new branch
5. Runs: `codex exec {plan}` or `claude -p {plan}` - Execute the selected implementation agent

**Timeout**: Uses `CODEX_TIMEOUT_SECONDS` (default 1800s = 30 min)

**Raises**: `RuntimeError` from shell execution if any step fails

### `has_changes() -> bool`
Checks if there are uncommitted changes in the repository.

**Uses**: `git status --porcelain`

**Returns**: True if any changes exist, False otherwise

### `push(branch_name: str, change_name: str, commit_type: str = "feat") -> str`
Commits and pushes changes to remote.

**Process**:
1. Validates branch name
2. Checks for changes using `has_changes()`
3. Raises error if no changes found
4. Runs: `git add .` - Stage all changes
5. Runs: `git commit -m "{commit_type}: {change_name}"` - Commit with message
6. Runs: `git push origin {branch_name}` - Push to remote
7. Runs: `git rev-parse HEAD` - Get committed SHA

**Returns**: Commit SHA (head revision)

**Commit Types**:
- "feat" - For features (default)
- "fix" - For bugfixes
- Custom: Any string (conventional commits style)

**Raises**: `RuntimeError` if no changes to commit

## Pull Request Management

### `create_pull_request(branch_name: str, change_name: str, body_text: str, title_type: str = "feat", body_label: str = "Plan") -> PullRequest`
Creates a pull request on GitHub.

**Process**:
1. Ensures GitHub is configured
2. Validates branch name
3. Creates PR payload:
   - title: "{title_type}: {change_name}"
   - head: feature branch name
   - base: GITHUB_BASE_BRANCH (usually "main")
   - body: Formatted with generation note and label
   - maintainer_can_modify: true
4. POSTs to GitHub API
5. Extracts PR data

**PR Body Format**:
```
Generated by Channel Cast Agent.

{body_label}:

{body_text}
```

**Returns**: `PullRequest` dataclass with:
- `number`: PR number
- `url`: HTML link to PR
- `head_sha`: Commit SHA of PR head

**Raises**: GitHub API errors from `github_request()`

## Configuration Dependencies

- `CODEX_TIMEOUT_SECONDS`: Timeout for Codex execution
- `GITHUB_BASE_BRANCH`: Base branch for PRs (usually "main")
- `GITHUB_REPOSITORY`: Target repository (owner/repo)

## API Endpoints Used

- `POST /repos/{owner}/{repo}/pulls` - Create pull request

## Usage Pattern

```python
from ai_agent.workflow import (
    slugify_branch_name,
    implement,
    push,
    create_pull_request
)

# Generate branch name
branch = slugify_branch_name("Add offline support")  # "feature/add-offline-support"

# Run Codex implementation
implement(plan_prompt, branch)

# Commit and push changes
sha = push(branch, "Add offline support", "feat")

# Create pull request
pr = create_pull_request(
    branch,
    "Add offline support",
    plan_text,
    "feat",
    "Plan"
)
print(f"PR created: {pr.url}")
```

## Error Handling

**Validation errors**:
- Invalid branch names raise ValueError

**Execution errors**:
- Git/Codex failures raise RuntimeError from shell
- GitHub API errors raise RuntimeError from github_request
- No changes to commit raises RuntimeError

**All errors are caught by Telegram bot and reported to user**
